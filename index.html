MR Review
How to Use
1
Configure Settings

Click the gear icon and fill in your GitLab Instance URL, Project Path (e.g. namespace/project), and a Personal Access Token with read_api scope. All three fields are required.

2
Enter MR ID & Fetch

Type the Merge Request ID (the number after !) and press Fetch or hit Enter.

3
Run Gunther

Click Run Gunther to manually post @gunther pendingreview on the MR, wait 12 seconds for the bot to generate the assignment, then automatically fetch and display the latest reviewers. Use this when you want to force a fresh assignment regardless of any existing review comment.

4
Auto-trigger Review Assignment

If the latest comment doesn't contain reviewer details, the app automatically posts @gunther pendingreview and waits 12 seconds for the bot to generate the assignment, then re-fetches.

5
Already Reviewed

Reviewers who gave a 👍 reaction directly on the MR appear here. They are automatically excluded from the Enough Reviewers calculation.

6
Enough Reviewers

The minimum set of reviewers needed to cover every pending file, computed using a greedy algorithm across primary and backup reviewers. Click ✕ on any chip to exclude that reviewer and recompute instantly.

7
Removed from Consideration

Reviewers you exclude via ✕ appear in a separate Removed from consideration row below the chips. Click + next to any name to re-add them — the minimum reviewer list recomputes immediately. A reviewer cannot be removed if doing so would leave any file with no eligible reviewer.

8
Ask for Review

Click the chat icon in the Enough Reviewers header to open the Ask for Review panel. It pre-fills a message — Hi, Please review this [MR link] — that you can Copy and share with reviewers. Use the Zoho Cliq icon beside the copy button to open Cliq directly.

9
Copy JSON

Click Copy JSON in the Ask for Review panel to copy a structured JSON payload to your clipboard. The object contains:

  • not_reviewed_by — usernames of the minimum reviewers still needed (respects any reviewers you excluded via ✕)
  • message — the review request text as currently shown in the message box (including any edits you made)
  • mr_title — the MR number and title (e.g. MR !147082 — Fix …)
  • mr_link — direct URL to the merge request
10
Review Comment

The latest comment is parsed into file sections — Needs Approval (orange), Backup Approved (yellow), and Approved (blue) — each listing its files with primary and backup reviewers.

Tips
  • Use the moon/sun button to toggle dark and light theme. Your preference is saved.
  • Pick an accent colour from Settings → Accent Color to personalise the app's look.
  • A red dot on the gear icon means a settings field is empty.
  • Defaults can be changed in config.js without touching app logic.

  4. Click Create personal access token and copy the token
